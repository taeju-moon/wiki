# W3 - Hadoop 멀티노드 클러스터 (Docker)

master 1대, worker 2대로 구성한다. replication은 2로 잡는다.

- master: NameNode, ResourceManager
- worker1, worker2: DataNode, NodeManager

## 폴더 구조

```
missions/W3/
├── base/Dockerfile          # Java + Hadoop + SSH 설치
├── config/                  # master/worker 공용 설정 (core-site, hdfs-site, mapred-site, yarn-site, workers)
├── master/                  # Dockerfile + entrypoint.sh
├── worker/                  # Dockerfile + entrypoint.sh
└── docker-compose.yml
```

## 빌드

```
cd missions/W3
docker build -t hadoop-base:3.3.6 -f base/Dockerfile base
docker compose build
```

base 이미지는 compose가 자동으로 안 만들어주므로 먼저 따로 빌드해야 한다.

## 실행

```
docker compose up -d
```

master가 뜨고 나서 worker가 붙는다. worker entrypoint에 master:9000 대기 로직이 있어서 순서 안 맞아도 알아서 기다린다.

## 종료

```
docker compose down       # 컨테이너만 삭제, 볼륨(데이터)은 남음
docker compose down -v    # 볼륨까지 삭제, 완전 초기화
```

## 상태 확인

```
docker exec master jps      # NameNode, ResourceManager 떠야 함
docker exec worker1 jps     # DataNode, NodeManager 떠야 함
docker exec worker2 jps
```

- HDFS 웹 UI: http://localhost:9870
- YARN 웹 UI: http://localhost:8088

## HDFS 사용법

```
docker exec master hdfs dfs -mkdir -p /user/root
docker exec master hdfs dfs -mkdir /input

docker cp sample.txt master:/tmp/sample.txt
docker exec master hdfs dfs -put /tmp/sample.txt /input/sample.txt

docker exec master hdfs dfs -ls /input
docker exec master hdfs dfs -cat /input/sample.txt
docker exec master hdfs dfs -get /input/sample.txt /tmp/downloaded.txt

docker exec master hdfs fsck /input/sample.txt -files -blocks -locations
```

fsck 결과에서 각 블록마다 DatanodeInfoWithStorage가 2개(worker1, worker2) 찍히면 정상이다.

## MapReduce 실행 (WordCount, Python Streaming)

내장 예제 jar 안 쓰고 직접 짠 mapper/reducer로 돌린다. 소스는 `mapreduce/mapper.py`, `mapreduce/reducer.py`. Hadoop Streaming이라 컴파일은 필요 없고, 컨테이너에 python3만 있으면 그대로 돈다.

입력 이북: [Pride and Prejudice - Project Gutenberg](https://www.gutenberg.org/cache/epub/1342/pg1342.txt), `mapreduce/input/ebook.txt`에 받아둔 상태.

### 1. 로컬에서 먼저 검증

```
cd missions/W3/mapreduce
cat input/ebook.txt | python3 mapper.py | sort | python3 reducer.py | head
```

여기서 정상 출력되는 것만 확인하고 넘어간다. (실제 셔플/정렬은 hadoop이 해주는거라 로컬 sort는 검증용일 뿐)

### 2. 소스/입력 파일 컨테이너로 복사 → HDFS 업로드

```
docker cp mapreduce/mapper.py master:/tmp/mapper.py
docker cp mapreduce/reducer.py master:/tmp/reducer.py
docker cp mapreduce/input/ebook.txt master:/tmp/ebook.txt

docker exec master hdfs dfs -mkdir -p /input
docker exec master hdfs dfs -put -f /tmp/ebook.txt /input/ebook.txt
docker exec master hdfs dfs -ls /input
```

### 3. Job 실행

```
docker exec master hadoop jar /opt/hadoop/share/hadoop/tools/lib/hadoop-streaming-3.3.6.jar \
  -files /tmp/mapper.py,/tmp/reducer.py \
  -mapper "python3 mapper.py" \
  -reducer "python3 reducer.py" \
  -input /input/ebook.txt \
  -output /output
```

output 디렉토리가 이미 있으면 에러 나므로, 재실행할 땐 `hdfs dfs -rm -r /output` 먼저 한다.

### 4. 결과 확인 + 로컬로 가져오기

```
docker exec master hdfs dfs -ls /output
docker exec master hdfs dfs -cat /output/part-00000 | sort -k2 -nr | head -20

docker exec master hdfs dfs -get /output /tmp/output
docker cp master:/tmp/output ./mapreduce/output
```

`part-00000`에 단어별 카운트 tab-separated로 찍힌다.

## 트러블슈팅

**`docker run IMAGE bash -c "..."`로 파일 확인하려다가 컨테이너가 안 꺼짐**
master/worker 이미지는 ENTRYPOINT가 `/entrypoint.sh`로 고정돼 있음. 뒤에 붙인 bash 명령이 검증용이 아니라 entrypoint.sh 인자로 들어가버려서 실제 데몬이 떠버림. 파일만 확인하고 싶으면 `--entrypoint bash`로 덮어써야 한다.

**컨테이너 재시작하면 데이터 날아가는 문제**
`hdfs namenode -format`을 매번 하면 메타데이터가 초기화됨. master entrypoint.sh에서 `/hadoop/dfs/name` 디렉토리가 비어있을 때만 format 하도록 체크함.

**YARN에서 멀쩡한 task가 죽는 경우**
Docker 컨테이너 안에서는 가상 메모리 사용량이 실제보다 부풀려져 보여서 YARN이 오판하고 task를 kill할 수 있음. yarn-site.xml에서 `pmem-check-enabled`, `vmem-check-enabled` 둘 다 false로 꺼둠.

**worker가 컨테이너는 떴는데 datanode가 안 뜸**
master의 NameNode가 완전히 준비되기 전에 worker가 datanode를 켜려고 하면 실패함. worker entrypoint.sh에 `/dev/tcp/master/9000` 연결될 때까지 기다리는 루프 넣음.
