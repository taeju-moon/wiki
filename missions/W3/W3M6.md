# W3 - Hadoop 멀티노드 클러스터 (Docker)

namenode 1대, worker 2대로 구성한다. replication은 2로 잡는다.

- namenode: NameNode, ResourceManager
- worker1, worker2: DataNode, NodeManager

## 폴더 구조

```
missions/W3/
├── base/Dockerfile          # Java + Hadoop + SSH 설치
├── config/                  # namenode/worker 공용 설정 (core-site, hdfs-site, mapred-site, yarn-site, workers)
├── master/                  # namenode 컨테이너용 Dockerfile + entrypoint.sh (컨테이너/호스트명은 namenode, 폴더명만 master로 남아있음)
├── worker/                  # Dockerfile + entrypoint.sh
├── mapreduce/               # MapReduce 소스 (mapper/reducer)
│   ├── mapper.py
│   ├── reducer.py
│   └── input/               # 입력 데이터 (git에는 포함 안 됨, 아래 "데이터셋" 참고)
└── docker-compose.yml
```

## 빌드

```
cd missions/W3
docker build -t hadoop-base:3.3.6 -f base/Dockerfile base
docker compose build
```

base 이미지는 compose가 자동으로 안 만들어주므로 먼저 따로 빌드해야 한다.

## 실행

```
docker compose up -d
```

namenode가 뜨고 나서 worker가 붙는다. worker entrypoint에 namenode:9000 대기 로직이 있어서 순서 안 맞아도 알아서 기다린다.

## 종료

```
docker compose down       # 컨테이너만 삭제, 볼륨(데이터)은 남음
docker compose down -v    # 볼륨까지 삭제, 완전 초기화
```

## 상태 확인

```
docker exec namenode jps    # NameNode, ResourceManager 떠야 함
docker exec worker1 jps     # DataNode, NodeManager 떠야 함
docker exec worker2 jps
```

- HDFS 웹 UI: http://localhost:9870
- YARN 웹 UI: http://localhost:8088

## HDFS 사용법

```
docker exec namenode hdfs dfs -mkdir -p /user/root
docker exec namenode hdfs dfs -mkdir /input

docker cp sample.txt namenode:/tmp/sample.txt
docker exec namenode hdfs dfs -put /tmp/sample.txt /input/sample.txt

docker exec namenode hdfs dfs -ls /input
docker exec namenode hdfs dfs -cat /input/sample.txt
docker exec namenode hdfs dfs -get /input/sample.txt /tmp/downloaded.txt

docker exec namenode hdfs fsck /input/sample.txt -files -blocks -locations
```

fsck 결과에서 각 블록마다 DatanodeInfoWithStorage가 2개(worker1, worker2) 찍히면 정상이다.

## MapReduce 실행 (상품별 평균 평점, Python Streaming)

### 데이터셋

[Amazon Reviews 2023](https://amazon-reviews-2023.github.io/)의 `All_Beauty` 카테고리 리뷰 데이터를 사용한다. UC San Diego McAuley Lab이 공개한 데이터셋으로, [Hugging Face(McAuley-Lab/Amazon-Reviews-2023)](https://huggingface.co/datasets/McAuley-Lab/Amazon-Reviews-2023)에서 카테고리별로 내려받을 수 있다. `All_Beauty` 카테고리 기준 약 70만 건의 리뷰가 JSON Lines(한 줄에 리뷰 하나, JSON 객체) 포맷으로 들어있다.

한 줄 예시:
```json
{"rating": 5.0, "title": "...", "text": "...", "asin": "B00YQ6X8EO", "parent_asin": "B00YQ6X8EO", "user_id": "...", "timestamp": 1588687728923, "helpful_vote": 0, "verified_purchase": true}
```

용량이 커서(약 320MB) git에는 커밋하지 않는다(`.gitignore`에 `missions/W3/mapreduce/input/aws.jsonl` 등록됨). 위 링크에서 받은 파일을 `missions/W3/mapreduce/input/aws.jsonl` 경로에 두면 아래 실행 과정을 그대로 따라할 수 있다.

### 목표

`parent_asin`(같은 상품의 색상/사이즈 등 variant를 하나로 묶는 상품 식별자, asin보다 상위 개념) 기준으로 리뷰 평점의 평균을 구한다. 소스는 `mapreduce/mapper.py`, `mapreduce/reducer.py` (combiner는 사용하지 않음).

Hadoop Streaming을 쓰기 때문에 **별도 컴파일 과정이 없다** — 컨테이너에 python3만 설치되어 있으면 스크립트 경로만 지정해서 바로 실행된다(base 이미지에 python3 포함).

`docker-compose.yml`에서 로컬 `./mapreduce` 디렉토리를 namenode 컨테이너의 `/mapreduce`에 그대로 마운트해두었기 때문에, 소스/입력 파일을 `docker cp`로 옮길 필요 없이 컨테이너 안에서 `/mapreduce/...` 경로로 바로 접근할 수 있다. 반대로 `hdfs dfs -get`으로 받은 결과도 `/mapreduce/output`에 쓰면 로컬 `missions/W3/mapreduce/output`에서 바로 확인 가능하다.

### 동작 방식

- **mapper.py**: 입력을 한 줄씩(리뷰 하나씩) JSON으로 파싱해 `parent_asin \t rating` 형태로 출력한다. JSON 파싱에 실패하는 줄, `parent_asin`/`rating` 필드가 없는 줄, rating이 숫자로 변환 안 되는 줄은 건너뛴다.
- **reducer.py**: Hadoop이 key(`parent_asin`) 기준으로 정렬해서 넘겨준다는 전제 하에, 같은 상품이 연속으로 들어오는 동안 합계/개수를 누적하다가 key가 바뀌는 시점에 `parent_asin \t 리뷰개수 \t 평균평점` 형태로 출력한다.

### 1. 로컬에서 먼저 검증

컴파일이 필요 없으므로 파이썬 인터프리터로 파이프라인을 그대로 이어서 실행해본다.

```
cd missions/W3/mapreduce
cat input/aws.jsonl | python3 mapper.py | sort | python3 reducer.py | head
```

여기서 정상 출력되는 것만 확인하고 넘어간다 (실제 셔플/정렬은 hadoop이 해주는거라 로컬 sort는 검증용일 뿐).

### 2. 입력 데이터를 HDFS에 업로드

`/mapreduce`가 이미 컨테이너에 마운트되어 있으므로 `docker cp` 없이 바로 `-put`할 수 있다.

```
docker exec namenode hdfs dfs -mkdir -p /input
docker exec namenode hdfs dfs -put -f /mapreduce/input/aws.jsonl /input/aws.jsonl
docker exec namenode hdfs dfs -ls /input
```

### 3. Job 실행

```
docker exec namenode hadoop jar /opt/hadoop/share/hadoop/tools/lib/hadoop-streaming-3.3.6.jar \
  -files /mapreduce/mapper.py,/mapreduce/reducer.py \
  -mapper "python3 mapper.py" \
  -reducer "python3 reducer.py" \
  -input /input/aws.jsonl \
  -output /output
```

output 디렉토리가 이미 있으면 에러 나므로, 재실행할 땐 `hdfs dfs -rm -r /output` 먼저 한다.

### 4. 결과 확인 + 로컬로 가져오기

```
docker exec namenode hdfs dfs -ls /output
docker exec namenode hdfs dfs -cat /output/part-00000 | head -20

docker exec namenode hdfs dfs -get -f /output /mapreduce/output
```

`/mapreduce`가 로컬 `./mapreduce`와 마운트되어 있으므로 `-get` 결과가 바로 로컬 `missions/W3/mapreduce/output`에 생긴다.

### 결과 해석 및 검증

- `part-00000`(리듀서가 여러 개면 `part-0000N`도 생김)에 `parent_asin \t 리뷰개수 \t 평균평점`이 tab으로 구분되어 찍힌다. 예: `B00YQ6X8EO\t12\t4.42`는 상품 B00YQ6X8EO에 리뷰 12건이 달렸고 평균 평점이 4.42라는 뜻.
- 리뷰가 많은/평점이 높은 상품 순으로 보기:
  ```
  sort -k2 -nr missions/W3/mapreduce/output/part-00000 | head -20   # 리뷰 개수순
  sort -k3 -nr missions/W3/mapreduce/output/part-00000 | head -20   # 평균 평점순
  ```
- 1단계 로컬 실행 결과와 비교해서 hadoop 결과가 맞는지 검증:

```
sort missions/W3/mapreduce/output/part-00000 > /tmp/hadoop_result.tsv
cat missions/W3/mapreduce/input/aws.jsonl | python3 missions/W3/mapreduce/mapper.py | sort | python3 missions/W3/mapreduce/reducer.py | sort > /tmp/local_result.tsv
diff /tmp/hadoop_result.tsv /tmp/local_result.tsv && echo "일치: hadoop 결과와 로컬 결과가 같다"
```

## 트러블슈팅

**`docker run IMAGE bash -c "..."`로 파일 확인하려다가 컨테이너가 안 꺼짐**
namenode/worker 이미지는 ENTRYPOINT가 `/entrypoint.sh`로 고정돼 있음. 뒤에 붙인 bash 명령이 검증용이 아니라 entrypoint.sh 인자로 들어가버려서 실제 데몬이 떠버림. 파일만 확인하고 싶으면 `--entrypoint bash`로 덮어써야 한다.

**컨테이너 재시작하면 데이터 날아가는 문제**
`hdfs namenode -format`을 매번 하면 메타데이터가 초기화됨. namenode entrypoint.sh(`master/entrypoint.sh`)에서 `/hadoop/dfs/name` 디렉토리가 비어있을 때만 format 하도록 체크함.

**YARN에서 멀쩡한 task가 죽는 경우**
Docker 컨테이너 안에서는 가상 메모리 사용량이 실제보다 부풀려져 보여서 YARN이 오판하고 task를 kill할 수 있음. yarn-site.xml에서 `pmem-check-enabled`, `vmem-check-enabled` 둘 다 false로 꺼둠.

**worker가 컨테이너는 떴는데 datanode가 안 뜸**
namenode가 완전히 준비되기 전에 worker가 datanode를 켜려고 하면 실패함. worker entrypoint.sh에 `/dev/tcp/namenode/9000` 연결될 때까지 기다리는 루프 넣음.
