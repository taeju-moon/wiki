# 위키피디아 GDP 데이터 ETL 파이프라인

이번 과제는 위키피디아의 **국가별 명목 GDP 데이터**를 가져와서, 깔끔하게 가공하고 DB에 적재해보는 심플한 미니 ETL 파이프라인 구현 과제이다.

## 폴더 구조

프로젝트 폴더 내에는 다음과 같은 파일들이 있습니다:

- `etl_project_gdp.py` : JSON 파일 적재를 목표로 하는 기본 ETL 스크립트.
- `etl_project_gdp_with_sql.py` : 기본 로직에 SQLite DB 연동과 SQL 분석 로직을 추가. 
- `Countries_by_GDP.json` : 첫 번째 스크립트 실행 후 데이터가 저장되는 JSON 결과물. (upload 제외)
- `World_Economies.db` : 두 번째 스크립트를 통해 생성되는 SQLite 데이터베이스 파일. (upload 제외)
- `etl_project_log.txt` : 로그 파일 (upload 제외)

## 이번 프로젝트를 하며 배운 점들

1. **웹 크롤링 시 HTML 구조 파악**
   BeautifulSoup으로 데이터를 뽑아올 때 단순히 태그만 긁어오는 게 아니라 HTML `<table>` 구조를 정확히 파악해야 한다는 것을 배웠다. 
   특히 최신 위키피디아 테이블에는 'UN Region' 컬럼이 사라져서 당황했는데, 이를 해결하기 위해 pycountry-convert 라이브러리를 통해 해당 열을 채워줄 수 있었다.

2. **Pandas를 활용한 기초 전처리**
   지저분했던 GDP 텍스트 데이터에서 콤마(,)를 떼고 실수형으로 바꾼 뒤, 10억(1B USD) 단위로 나누고 소수점 2자리로 맞춰서 정렬하는 과정을 경험했다. 

3. **객체지향으로 확장성 확보**
   `Extractor`, `Transformer`, `Loader` 로 역할을 쪼개서 클래스를 만들었더니 코드를 깔끔하게 했다.
   나중에 JSON 저장 방식에서 SQLite 저장 방식(`SQLLoader`)으로 업그레이드할 때, 전체 코드를 뜯어고칠 필요 없이 해당 클래스만 갈아끼우면 되어서 편했다.

4. **SQL 윈도우 함수로 복잡한 집계**
   "각 대륙(Region)별 최상위 5개 국가의 평균 GDP 구하기"라는 미션을 파이썬이 아닌 순수 SQL로 해결해야 했다. 처음엔 막막했지만, `WITH` 문으로 CTE를 만들고 `ROW_NUMBER() OVER (PARTITION BY ...)` 윈도우 함수를 써보면서, SQL만으로도 얼마나 강력하고 복잡한 집계가 가능한지 확실히 체감했다.
