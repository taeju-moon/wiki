# SQL 기초 튜토리얼 실습

이번 과제는 SQL Tutorial을 처음부터 끝까지 따라가며, Jupyter Notebook 환경에서 기본 SQL 명령어들을 직접 실행하는 과제다.

## 이번 과제를 하며 배운 점들

1. **순수 SQL 사용**
   평소엔 Pandas로 대충 처리하던 데이터 필터링이나 그룹화를 `SELECT`, `WHERE`, `GROUP BY`, `HAVING` 같은 순수 SQL 쿼리로만 처리했다. 막상 해보니 SQL만으로도 웬만한 데이터 가공이 가능했다.

2. **SQLite의 한계 체감 (RIGHT JOIN 미지원 등)**
   조인(JOIN) 실습을 하다가 에러가 터져서 확인해보니, SQLite는 `RIGHT JOIN`을 아예 지원하지 않았다. 어쩔 수 없이 주석 처리로 막아뒀는데, RDBMS마다 지원하는 SQL 문법이 은근히 다르다는 걸 뼈저리게 배웠다. 이 밖에도 여러 구문이 sqlite에서는 지원되지 않았다.

3. **다른 문법으로 우회하기 (CREATE TABLE AS)**
   다른 DB에서 흔히 쓰이는 `SELECT INTO` 같은 문법이 SQLite에선 안 먹혔다. 그래서 `CREATE TABLE ... AS SELECT ...` 구문으로 테이블을 복사해서 백업 테이블(`US_Customers_Backup` 등)을 만들었다. 안 되는 건 어떻게든 다른 문법으로 우회해서 해결하는 요령이 생겼다.


## SQL문법 요약

- **SELECT / FROM**: 어디서 무엇을 가져오는가
- **WHERE**: 가져올 때의 조건
- **GROUP BY / HAVING**: 기준으로 묶고, 이미 그룹화한 이후 조건문 걸기
- **ORDER BY**: 결과 정렬
- **JOIN**: 여러 테이블 합치기
- **UNION**: 쿼리 결과 합치기, 중복 제거됨 (UNION ALL 중복 제거 안함)
- **INSERT / UPDATE / DELETE**: 데이터 삽입 / 수정 / 삭제
